# cas (Claude Account Switcher) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Note:** Per user instructions, this plan contains NO implementation code — only precise behavioral specifications, exact paths, and exact commands. Spec: `docs/superpowers/specs/2026-06-10-cas-design.md`.

**Goal:** A sourced zsh tool that switches Claude Code accounts per-terminal via `CLAUDE_CONFIG_DIR` profile dirs that are symlink views onto canonical `~/.claude`.

**Architecture:** One zsh file (`cas.zsh`) defining a `cas` dispatcher function plus `_cas_*` helpers and tab completion. Profiles live in `$HOME/.claude-profiles/<name>/`. All paths derive from `$HOME`, so tests run the tool inside a sandbox `HOME` via fresh `zsh -f` invocations.

**Tech Stack:** Pure zsh + jq. Test harness is a plain zsh script with assert helpers; no test framework.

---

## File Structure

- `cas.zsh` — entire tool: dispatcher, helpers, completion registration.
- `test/run_tests.zsh` — test harness + all tests. Creates a temp sandbox `HOME` per test containing a fake `~/.claude` (with `settings.json`, `CLAUDE.md`, `plugins/` dir, `debug/` dir, `history.jsonl`) and a fake `~/.claude.json` (with `mcpServers` and `oauthAccount.emailAddress` keys). Runs each test body in a child `zsh -f -c` that sources `cas.zsh`, so env exports don't leak between tests. Exits non-zero on any failure, prints PASS/FAIL per test.
- `README.md` — install (source line for `~/.zshrc`) and usage.

## Shared behavioral contract (all tasks reference this)

- Canonical dir: `$HOME/.claude`. Canonical account file: `$HOME/.claude.json`. Profiles root: `$HOME/.claude-profiles`.
- **Denylist** (never symlinked; per-profile if they exist): `.claude.json`, `.credentials.json`, `debug`, `.last-update-result.json`, `.last-cleanup`, `stats-cache.json`, `mcp-needs-auth-cache.json`.
- **Link set** = every top-level entry of `$HOME/.claude` minus the denylist, computed at runtime (no hardcoded list). Links are absolute symlinks into `$HOME/.claude`.
- **Valid profile name**: matches `^[A-Za-z0-9_-]+$` and is none of: `default`, `add`, `rm`, `heal`. Invalid names → error message to stderr, return 1.
- All `.claude.json` writes by cas go via temp file + `mv` in the same directory.
- `cas` must remain a shell *function* (it mutates the calling shell's environment); helpers prefixed `_cas_` to avoid namespace pollution.

2. `cas default` afterwards: `CLAUDE_CONFIG_DIR` unset/empty, `CAS_PROFILE` equals `default`.

4. **mcpServers sync:** mutate canonical `.claude.json`'s `mcpServers` after `add`, then `cas work` → profile `.claude.json` `mcpServers` matches new canonical value, and its other keys (pre-seeded `oauthAccount` stub) survive untouched.

6. **Heal check warning:** replace profile's `settings.json` symlink with a regular file, then `cas work` → switch still succeeds but stderr names `settings.json` as forked and mentions `cas heal`.
